# Repo Release Manager — Spec Kit Prompts

Consolidated prompts for kicking off the Repository Release Management Platform with Spec Kit and Claude Code.

## How to use

1. Initialise Spec Kit in your repo:
   ```bash
   uvx --from git+https://github.com/github/spec-kit.git specify init repo-release-manager
   cd repo-release-manager
   ```
   Choose Claude Code as the AI assistant.

2. In Claude Code, run the slash commands in order, pasting the matching prompt file as input:

   | Order | Command | File |
   |-------|---------|------|
   | 1 | `/constitution` | `01-constitution.md` |
   | 2 | `/specify` | `02-specify.md` |
   | 3 | `/clarify` | (answer any questions; everything is in the spec) |
   | 4 | `/plan` | `03-plan.md` |
   | 5 | `/tasks` | use `04-tasks-guidance.md` to validate the milestone ordering |
   | 6 | `/analyze` | review consistency |
   | 7 | `/implement` | execute Milestone 1 first; review before continuing |

3. After each milestone, run its smoke test and only then proceed to the next.

## What's in scope for v1

- Azure DevOps Git provider (PAT auth)
- SQLite + EF Core 10
- Layered architecture, controllers + services, no CQRS
- Logical projects with many-to-many repo assignments
- Conventional commit parsing with Jira ticket extraction
- Per-repo and per-project change visibility
- Release notes generated by Handlebars templates, grouped by ticket
- Confluence Cloud publishing
- Jira Cloud reconciliation with three buckets
- Admin/Viewer roles via JWT
- React + Vite + shadcn frontend

## What's out of scope for v1

- GitHub, GitLab, Bitbucket (abstractions ready, no implementations)
- Multi-tenancy
- SSO
- Jira Data Center / Server
- Slack/Teams notifications
- CI/CD triggering
- Bidirectional Jira sync
