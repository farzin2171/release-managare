### `SetupKeyAuthorizationFilter` (IAsyncActionFilter — Api layer)

Located in `RepoManager.Api/Filters/`.

```csharp
public sealed class SetupKeyAuthorizationFilter : IAsyncActionFilter
{
    public async Task OnActionExecutionAsync(ActionExecutingContext context, ActionExecutionDelegate next)
    {
        // 1. Read X-Setup-Key header from context.HttpContext.Request.Headers
        // 2. Read configured key from IConfiguration["RELEASE_MANAGER_SETUP_KEY"]
        // 3. Compare using CryptographicOperations.FixedTimeEquals (constant-time)
        // 4. If mismatch → context.Result = new ObjectResult(new { code = "setup_key_invalid" }) { StatusCode = 401 }; return;
        // 5. Otherwise → await next()
    }
}
```

**Applied to** `AuthController.Setup` only via `[ServiceFilter(typeof(SetupKeyAuthorizationFilter))]`.

**Never logs the key value** — only logs `"Setup key validation failed"` (without the value) on mismatch at `Warning` level.

## Modified Service Interfaces

### `ITemplateService` (Application layer)

**Existing methods**: `GetAllAsync`, `GetByIdAsync`, `CreateAsync`, `UpdateAsync`, `DeleteAsync`

**Changes**:
- `UpdateAsync` MUST check `IsSystem` and throw `ForbiddenException("system_template_readonly")` before applying any changes.
- `DeleteAsync` MUST check `IsSystem` and throw `ForbiddenException("system_template_readonly")` before deleting.

**New method**:
```csharp
Task<ReleaseNoteTemplateDto> CloneAsync(int templateId, CancellationToken ct = default);
```

**Error contracts for `CloneAsync`**:
- `NotFoundException` — template with `templateId` not found.
- `ConflictException("clone_name_exhausted")` — 100 name suffixes already taken.

**Clone naming algorithm** (runs inside a DB transaction):
1. `baseName = $"{template.Name} (copy)"`
2. Query all names starting with `baseName` (case-insensitive prefix match).
3. Try `baseName`, then `baseName + " 2"` … `baseName + " 100"`.
4. First available name wins; insert new row with `IsSystem = false`.

### `IProjectService` (Application layer)

**Change to `CreateAsync`**: After persisting the new `Project` row, query for the system template by `IsSystem = true` and `Name = "Release Summary (Default)"`. If found, insert a `ProjectTemplateBinding` with:
- `Kind = TemplateBindingKind.ReleaseSummary`
- `SortOrder = 1`
- `TemplateId = <system template id>`

This runs within the same DB transaction as the project creation (existing cross-aggregate write pattern).

## Frontend Contracts

### Auth Store — Zustand additions

```typescript
interface AuthStoreAdditions {
  // New action
  scheduleRefresh: (accessToken: string) => void;
  // Internal timer ref (not exposed in store state)
  // _refreshTimer: ReturnType<typeof setTimeout> | null
}
```

`scheduleRefresh(accessToken)`:
1. Decode `exp` from the JWT payload using `atob`.
2. Cancel any existing `_refreshTimer`.
3. Compute `msUntilRefresh = exp * 1000 - Date.now() - 120_000`.
4. If `msUntilRefresh > 0`, set `_refreshTimer = setTimeout(() => triggerSilentRefresh(), msUntilRefresh)`.
5. Else, trigger the refresh immediately (token already within the 2-min window).

Called from:
- `login()` action — after receiving the access token.
- Store hydration (`onRehydrateStorage`) — on every app load that finds a persisted access token.
